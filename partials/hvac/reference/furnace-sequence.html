Purpose: Fast, defensible diagnosis for no-heat gas furnace calls by following sequence of operation instead of guessing.

Normal Sequence of Operation

  1. Thermostat calls for heat (W energized).
  2. Control board starts inducer motor.
  3. Pressure switch proves draft.
  4. Ignition source energizes (HSI or spark).
  5. Gas valve opens.
  6. Burners light; flame sensor proves flame.
  7. Indoor blower starts after timed delay.
  8. System satisfies thermostat, burners shut off, blower off-delay runs.

No-Heat Diagnostic Workflow

  1. Read fault code first (board LED pattern).
  2. Confirm thermostat call at board terminals (R to W).
  3. Verify line voltage and control voltage integrity.
  4. Step through sequence and identify exact break point.
  5. Perform confirming meter test before replacing any part.

Break Point → Probable Cause → Confirming Test

Break Point Likely Causes Confirming Test
Inducer does not start No call, board output failure, inducer motor failure, power issue Check W call, board inducer output voltage, motor amp/ohm, line voltage
Pressure switch won’t close Vent blockage, condensate issue, cracked/plugged tubing, weak inducer Measure draft/pressure, inspect venting/tubing/trap, verify switch continuity at proper draft
Igniter not glowing/sparking Failed igniter, no board output, wiring fault Check igniter resistance/continuity, verify ignition voltage output
Gas valve not opening No valve command, valve failure, safeties open Verify valve coil voltage during trial, check safeties in series
Flame lights then drops out Weak flame signal, dirty sensor, poor ground/polarity Measure flame rectification microamps, clean sensor, verify grounding and polarity
Burners run but blower delayed/absent Blower motor/capacitor/ECM issue, board fan output issue Verify fan output voltage, motor amp draw, capacitor µF, speed tap/profile
High-limit trips Low airflow, dirty filter/coil, undersized ducts, high temp rise Measure temp rise vs nameplate, static pressure, inspect airflow path

Quick Tips (New + Veteran Techs)

  • Fault code is a starting point, not final diagnosis.
  • Flame sensor issues are often signal/ground quality—not just “dirty sensor.”
  • Always verify airflow before condemning limit switches.
  • Capture readings (voltage, amps, µA flame signal, temp rise) for callback defense.

Closeout Data to Record

  • Fault code observed
  • Break point in sequence
  • Confirming test result
  • Repair performed
  • Final heat cycle verification (multiple cycles)

Rule: “Measure, then replace.” Parts swapping kills margin and callback rate.
